The system should allow users to create, view, edit, and manage tasks in an organized way using a Kanban board layout. Tasks must be categorized by status (To Do, In Progress, Done, Backlog) with drag-and-drop functionality for easy workflow management. The application should support filtering by priority and date, as well as pagination for better performance. Data must be securely stored and retrieved using a backend API with MongoDB as the database.

The system should enable organizations to track and manage IT assets such as hardware and software efficiently. It must allow users to add, update, and assign assets to employees based on their roles. The application should provide reporting features to monitor asset status, usage, and maintenance schedules. Data should be stored securely and accessible through a user-friendly interface.
